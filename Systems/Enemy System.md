### Purpose
- Defines enemy data, state, and behavior used during combat
- Provides the structure required for [[Enemy Phase]] and [[Enemy Intent]]

## Core Rule
- Enemies do not act directly; they execute their current intent to generate [[Action]]s

## Data

id
- enemy identifier

hp
- current health

max_hp
- maximum health

block
- current block value

statuses
- collection of [[Status]] instances applied to the enemy

alive
- boolean state (true if hp > 0)

position
- index in enemy order (used for left → right resolution)

intent
- current intent to execute this turn

next_intent
- intent determined for the next turn

intent_pattern
- rules or sequence used to determine next intent

## Rules

- An enemy exists in combat until hp <= 0
- Enemies are resolved in position order during [[Enemy Phase]]
- Each enemy executes exactly one intent per turn
- Enemies cannot act if not alive

## Order of Operations

During [[Enemy Phase]] for each enemy:

1. Check alive state
2. Resolve start-of-turn effects (enemy-owned)
3. Execute current intent
4. Determine next_intent
5. Resolve end-of-turn effects (enemy-owned)

## Intent Execution

- intent defines what actions will be generated
- execution produces [[Action]]s
- actions are enqueued into [[Action Queue]]

Example:

intent: attack (value = 8)

→ enqueue damage action targeting player

## Intent Storage

- intent is used during current turn
- next_intent is determined after execution
- next_intent becomes intent at the start of next enemy phase

## Interaction

- Uses [[Targeting]] to resolve targets for intent
- Generates [[Action]]s for [[Action Queue]]
- Modified by [[Status Effects]] during execution

## Edge Cases

Enemy dies before acting:
- skip execution
- no intent is executed

Enemy dies during its own action:
- current action resolves
- no further actions or intent updates

Enemy spawned mid-combat:
- must be assigned position
- must receive initial intent before next enemy phase

## Standardization Rules

- Enemy behavior is defined through intents, not direct logic
- All effects generated by enemies must go through [[Action Queue]]
- Intent must be fully defined before execution
